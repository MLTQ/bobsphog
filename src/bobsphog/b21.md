# `b21.py`

## Purpose

Runs the B2.1 systems upper bound on the real Qwen3.6 checkpoint. It compares
live future-aware retention, trace-driven token prefetch, and a pinned
prompt-level decode union while forcing the exact B2 output path.

## Components

### `RecordedRoute` / `load_recorded_route`

- **Does**: Reconstructs the prompt, selected token IDs, prefill groups, and
  per-token/per-layer expert requests from a B2 JSON artifact.
- **Rationale**: One canonical trace drives eviction, prefetch, and exact route
  validation without running a separate routing approximation.
- **Prompt union**: Deduplicates future pages by layer so the full sequence
  bundle can be transferred with bounded staging.

### `summarize_prefetch_pipeline`

- **Does**: Reports observed serial time, compute-only rate, and an ideal
  one-token-lookahead pipeline where prefetch for token `i+1` overlaps compute
  for token `i`.
- **Rationale**: Synchronous prefetch measures real work; the overlap estimate
  shows whether background execution is worth implementing next.

### `_run_control`

- **Does**: Executes one capacity/policy pair with real recurrent state, exact
  expert tensors, per-step prefetch and demand-cache deltas, synchronized
  timing, and offline-optimal miss validation.

### `run_b21`

- **Does**: Sweeps 1,280, 2,048, and 2,560 pages across oracle retention,
  token-prefetch, and an explicitly requested prompt-union control.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| CLI | `--trace` is an unmodified B2 result with routing data | Removing `routing_trace` from B2 |
| Capability check | Every run uses the recorded selected token IDs | Independent generation |
| Miss validation | Retention demand misses equal grouped Belady simulation | Different eviction timing |
| Pipeline estimate | Prefetch samples precede their aligned compute samples | Cross-token batching |
| Prompt union | Decode union plus largest prefill group fits simultaneously | Capacity below the reported requirement |

## Measurement caveats

Token prefetch is deliberately synchronous. Its serial rate is directly
measured; compute-only and ideal-overlap rates are upper bounds derived from the
same measured operations. Perfect future routes are unavailable online and will
be replaced by a learned prompt/early-state predictor only if this control shows
enough systems headroom.

The prompt-union policy front-loads and pins every unique decode page. Its
prefetch time is included in TTFT, while its prefill and decode forwards reveal
whether a predicted sequence bundle can eliminate later page faults entirely.
