# `a5.py`

## Purpose

Runs the physical A5 benchmark on CUDA with a scaled directly factorized model,
bounded prompt page cache, disjoint working-set churn, and transfer/compute
overlap.

## Components

### `A5Config`

- **Does**: Defines model scale, dtype, cache working set, and timing iterations.

### `run_a5`

- **Does**: Measures fully resident and physically paged execution, validates
  logits, switches working sets, and directly compares a synchronous cold cycle
  against scheduling the same cold pages concurrently with model execution.
- **Interacts with**: `PhysicalPageCache`, `PageCatalog`, and `ToyTransformer`.

### `main`

- **Does**: Exposes `bobsphog-a5` with device, dtype, working set, and timing
  controls.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| A5 report | Fully resident and paged paths execute the same selected pages | Different reference plan |
| Correctness | `max_absolute_logit_error` compares exact factor copies | Quantized/compressed cache |
| Memory result | CUDA `memory_allocated` is sampled after offload/empty-cache | Using only logical bytes |
| Churn result | Primary and secondary page sets are disjoint | Overlapping benchmark sets |
| Overlap result | Both cold cycles fetch the same page set | Different page plans or cache state |

## Notes

The benchmark model is randomly factorized because A5 tests systems mechanics,
not language quality. It avoids a large dense SVD while exercising the same page
execution contracts as trained models.

End-to-end cold-cycle timing uses host wall time through a final CUDA
synchronization. It includes scheduling, H2D transfer, and forward execution,
so the saved fraction measures exposed latency rather than submission time.
