# `b2.py`

## Purpose

Measures the Qwen3.6 decode working set over a fixed token path while changing
the number of exact expert pages resident on the accelerator. It joins measured
latency and transfer counters with trace-driven cache-policy controls.

## Components

### `TracingCudaExpertCache`

- **Does**: Records each layer's atomic expert request before delegating to the
  production CUDA cache.
- **Invariant**: Tracing must not reorder, add, or remove expert requests.

### `analyze_decode_working_sets`

- **Does**: Reports cumulative unique pages, new pages per decode forward,
  whole-model and per-layer overlap, and the recall/Jaccard achieved by using
  the previous token's working set as a one-step predictor.

### `_run_capacity`

- **Does**: Loads one paged model, times prompt prefill and synchronized manual
  decode, captures per-step cache deltas, and preserves the real recurrent/KV
  state.
- **Rationale**: Every capacity follows the same selected token IDs so routing,
  quality, and timing remain comparable.

### `run_b2`

- **Does**: Sweeps cache budgets, checks exact route/output agreement, replays
  the common trace through grouped LRU and offline-optimal replacement, and
  emits a compact routing trace for later predictor experiments. Policy results
  report both whole-run and decode-only miss/transfer bounds.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| Capacity comparison | All runs consume one fixed token path | Independent sampling |
| Route trace | One cache schedule call per MoE layer per forward | Batched scheduling |
| Cache simulation | Equal-size 6 MiB expert pages | Variable-size paging |
| Timing | CUDA synchronization bounds every forward | Unbounded asynchronous timing |

## Measurement caveats

The default sweep runs large caches before small caches. Any operating-system
file-cache warming therefore benefits the later small-cache runs, making the
measured speed advantage of larger caches conservative. Algorithmic miss and
byte counts are independent of host-cache warmth.

Offline-optimal replacement knows the complete future trace. It is a lower
bound on misses, not an achievable online controller and not proof that all I/O
can be overlapped with compute.
