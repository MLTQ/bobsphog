# `a4.py`

## Purpose

Orchestrates the harder compositional task, A2/A3 preparation, sparse signed
domain relationship graphs, and equal-budget A4 bundle comparisons.

## Components

### `default_a4_model_config` / `A4Config`

- **Does**: Defines the non-saturating modulus-8 task, larger toy model, graph,
  estimator, and evaluation settings.

### `run_a4`

- **Does**: Trains teacher/student/estimator, builds calibration-only graphs for
  both domains, and reports graph and policy metrics against independent,
  random, static-SVD, and oracle controls.
- **Interacts with**: Compositional task, relationship graph, retriever, and A4
  evaluation modules.

### `main`

- **Does**: Exposes `bobsphog-a4` with expensive step counts configurable.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| CLI users | Standard output is one JSON document | Logging to stdout or schema changes |
| A4 report | Graph and learned-graph gains are paired against their independent controls | Summary definition changes |
| Reproducibility | CPU run is deterministic for one seed | RNG sequencing changes |

## Notes

The graph is selected by the known synthetic domain. It is an offline index
experiment, not yet a learned query-to-graph router.
