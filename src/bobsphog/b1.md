# `b1.py`

## Purpose

Runs the first end-to-end pretrained smoke test with the Qwen3.6 language
scaffold resident on the 4090 and all routed experts file-backed.

## Components

### `B1Config`

- **Does**: Defines checkpoint path, CUDA device, expert-cache page capacity,
  seed text, and number of reported next-token candidates.

### `run_b1`

- **Does**: Loads the non-expert scaffold through the meta-device loader,
  tokenizes one input token, runs a cold forward that faults routed experts from
  NVMe, repeats the same forward from the warm cache, advances one token through
  Qwen's recurrent/KV state, and reports memory, transfer, cache, latency,
  parity, next-token, and decode working-set-overlap metrics.

### `main`

- **Does**: Exposes the experiment as `bobsphog-b1`.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| Working-set result | One-token routing needs at most `40 * 8 = 320` pages | Batched or multi-token default input |
| Warm replay | Cache capacity defaults to the entire one-token working set | Smaller default capacity |
| Checkpoint safety | All files already exist locally | Implicit network download during run |
| Output check | Cold and warm paths use identical input and weights | Sampling or stochastic model mode |

## Notes

This smoke test establishes executable out-of-core inference, not throughput.
The source loader is synchronous and expert execution uses the reference Python
loop. Prefetch prediction, background NVMe reads, and fused expert kernels come
after correctness and residency are demonstrated.
