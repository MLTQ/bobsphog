# `a2.py`

## Purpose

Orchestrates the complete A2 experiment from dense teacher training through
paged conversion, multi-budget student training, and page-utility analysis.

## Components

### `default_model_config` / `A2Config`

- **Does**: Defines the reproducible CPU-scale experiment and its loss/budget
  settings.
- **Rationale**: The default freezes resident tensors during student training to
  prevent capacity collapse into the skeleton.

### `resolve_device`

- **Does**: Validates explicit CPU, MPS, or CUDA selection and supports `auto`.

### `run_a2`

- **Does**: Trains teacher and student and returns JSON-compatible curves,
  metrics, and specialization measurements.
- **Interacts with**: Conversion, training, evaluation, and synthetic-task
  modules.

### `main`

- **Does**: Exposes the `bobsphog-a2` CLI with step, batch, device, and seed
  controls.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| CLI users | Standard output is one JSON document | Logging to stdout or schema changes |
| Future plots | Initial and trained curves use identical evaluation seeds | Evaluation protocol changes |
| Reproducibility | CPU runs with the same seed are deterministic | RNG sequencing changes |
| Anti-collapse baseline | Resident student tensors are frozen unless `--train-resident` is passed | Default trainability changes |

## Notes

The specialization summary is diagnostic. Random page dropout does not itself
guarantee human-legible domain experts; learned retrieval comes in A3.
