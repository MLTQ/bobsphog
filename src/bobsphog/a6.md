# `a6.py`

## Purpose

Runs the first end-to-end learned physical-paging experiment: train the A2/A3
toy model and counterfactual retriever, choose one complete working set from the
resident skeleton, then execute that plan through the bounded CUDA cache.

## Components

### `A6Config`

- **Does**: Defines training, utility-data, retrieval-budget, and CUDA settings.

### `run_a6`

- **Does**: Trains the teacher, paged student, and utility estimator; selects
  domain-conditioned prompt plans without executing optional pages; captures
  resident reference results; offloads every optional factor; and verifies that
  asynchronously fetched physical pages reproduce the selected-model logits.
- **Interacts with**: `learned_base_query_selection`, `PhysicalPageCache`, A2
  training, and A3 counterfactual labels.
- **Measurement note**: Warms the selector once before timing either domain so
  CUDA context/kernel initialization does not bias the first plan.

### `main`

- **Does**: Exposes `bobsphog-a6` with reduced-step controls for wiring checks.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| Prompt paging | Selection runs from the base-only state before cache attachment | Executing candidates during selection |
| Physical parity | Cached factors are exact copies of the trained page factors | Quantization or compression |
| Switch metrics | Cache capacity equals one selected working set | Capacity large enough for both plans |
| Domain comparison | Addition and multiplication use held-out prompt batches | Reusing utility-label batches |

## Notes

This experiment establishes integration, not a useful memory scale. Its pages
are deliberately tiny so the learned mechanics can be trained quickly. A5
remains the appropriate systems benchmark for transfer latency and VRAM savings.
