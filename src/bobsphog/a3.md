# `a3.py`

## Purpose

Orchestrates A2 model preparation, direct counterfactual label collection,
utility-estimator training, and equal-budget A3 selector comparisons.

## Components

### `A3Config`

- **Does**: Defines model training, utility sampling, estimator, and selection
  evaluation sizes.

### `run_a3`

- **Does**: Produces JSON-compatible training diagnostics, utility statistics,
  selector rows, aggregate gains over random/static baselines, and oracle regret.
- **Interacts with**: A2 training, catalog, utility data, retriever, and A3
  evaluation modules.

### `main`

- **Does**: Exposes `bobsphog-a3` with the expensive step counts configurable.

## Contracts

| Dependent | Expects | Breaking changes |
|-----------|---------|------------------|
| CLI users | Standard output is one JSON document | Logging to stdout or schema changes |
| A3 interpretation | Oracle/random/learned comparisons use equal page counts | Selector budget changes |
| Reproducibility | CPU execution is deterministic for one seed | RNG sequencing changes |

## Notes

The A3 estimator is intentionally small and domain-prefix driven. Success here
validates learnable selection mechanics, not general semantic retrieval.
